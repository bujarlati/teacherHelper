# teacherHelper 设计规格

日期：2026-06-15

## 目标

teacherHelper 是一个面向老师的 Windows 桌面应用，帮助老师整理教学备案，并把小学/初中数学应用题转成形象、可交互的课堂演示网页。

第一版采用“备课工作台 + 解题演示生成器”的组合形态：

- 老师输入教学知识点后，应用生成完整教案，包含教学目标、重点、难点、疑点、课堂导入、板书建议、例题、示例解法、课堂提问、作业建议、视频脚本、视频提示词和自动生成的视频素材。
- 老师输入具体数学应用题后，应用生成本地可交互网页，并通过 localhost 打开。路程、速度、时间类题目和方程题使用高质量专用互动模板；工程、几何等常见题型使用简版互动模板。
- API Key、模型名称、历史记录、导出文件和演示网页都保存在本机，不做账号系统和云同步。

## 非目标

第一版不做以下内容：

- 多用户账号、云端同步、权限管理。
- 全学科通用深度互动题目生成。
- 复杂视频剪辑、视频后期编辑或视频资产长期云存储。
- macOS 或 Linux 桌面打包。
- 直接运行模型生成的任意 JavaScript 代码。

## 技术路线

推荐实现为 Electron + React + Node。

- Electron 提供 Windows 桌面壳、窗口管理、本地文件访问和安全 IPC。
- React 提供备课工作台、题目演示、历史记录和设置页。
- Node 服务层负责硅基流动 API 调用、本地配置读写、结构化数据校验、Word 导出、演示网页生成、localhost 演示服务。

这条路线优先满足 Windows 桌面第一版的开发速度和本地能力：文件生成方便、localhost 演示页容易启动、Word 导出和 API 调用都能放在 Node 层统一处理。

## 硅基流动 API 依据

官方文档确认以下接口：

- 文本模型使用 `POST https://api.siliconflow.cn/v1/chat/completions`，请求包含 `model` 和 `messages`，鉴权头为 `Authorization: Bearer <API_KEY>`。参考：https://docs.siliconflow.cn/cn/api-reference/chat-completions/chat-completions
- 视频生成使用 `POST https://api.siliconflow.cn/v1/video/submit`，请求包含 `model`、`prompt` 等参数，响应返回 `requestId`。参考：https://docs.siliconflow.cn/cn/api-reference/videos/videos_submit
- 视频结果使用 `POST https://api.siliconflow.cn/v1/video/status`，请求包含 `requestId`，状态包含 `Succeed`、`InQueue`、`InProgress`、`Failed`，成功时返回视频 URL。参考：https://docs.siliconflow.cn/cn/api-reference/videos/get_videos_status
- 模型列表使用 `GET https://api.siliconflow.cn/v1/models`，可按 `type=text`、`type=video` 或 `sub_type` 筛选。参考：https://docs.siliconflow.cn/en/api-reference/models/get-model-list

设置页需要支持两组配置：

- 文本模型：API Key、模型名，用于教案生成和题目演示方案生成。
- 视频模型：API Key、模型名，用于视频生成。

## 主界面

主界面是备课工作台，左侧固定导航，右侧展示当前任务。

导航项：

- 今日备课
- 题目演示
- 历史记录
- 设置

今日备课页：

- 输入教学知识点。
- 生成结构化教案。
- 分块展示教案内容。
- 展示视频脚本、视频提示词和视频生成状态。
- 支持复制 Markdown。
- 支持导出 Word `.docx`。

题目演示页：

- 输入一道小学/初中数学应用题。
- 展示题型识别结果和解题步骤。
- 生成本地演示网页。
- 支持打开 localhost 演示页。
- 支持重新生成或退回简版图文页。

历史记录页：

- 保存每次生成的教案。
- 保存视频任务状态、视频 URL、提示词和脚本。
- 保存 Word 导出路径。
- 保存演示页项目路径。
- 支持继续编辑、重新导出、重新打开演示页。

设置页：

- 管理文本模型配置。
- 管理视频模型配置。
- 测试连接。
- 拉取模型列表。
- 清空本地配置。

## 教案生成流程

1. 老师输入知识点。
2. 应用读取本地文本模型配置。
3. Node 服务层调用硅基流动 `chat/completions`。
4. 系统提示要求模型返回固定 JSON。
5. 应用使用 schema 校验 JSON。
6. 校验通过后保存教案记录。
7. React 分块展示教案。
8. 应用生成 Markdown 内容。
9. 应用可导出 Word `.docx`。
10. 应用自动提交视频生成任务，同时保留可编辑的视频脚本和提示词。

教案 JSON 必须包含：

- `title`
- `grade_suggestion`
- `teaching_goals`
- `key_points`
- `difficult_points`
- `common_confusions`
- `lesson_flow`
- `board_design`
- `example_questions`
- `worked_solutions`
- `classroom_questions`
- `homework_suggestions`
- `video_script`
- `video_prompt`
- `markdown`

## 视频生成流程

视频生成不阻塞教案阅读。

1. 教案生成成功后，应用用 `video_prompt` 调用 `video/submit`。
2. 应用保存 `requestId` 和任务状态。
3. 后台轮询 `video/status`。
4. 状态为 `InQueue` 或 `InProgress` 时，页面显示排队或生成中。
5. 状态为 `Succeed` 时，保存视频 URL、推理耗时和 seed。
6. 状态为 `Failed` 时，保留脚本和提示词，允许老师修改后重新生成。

由于视频 URL 有时效，应用需要在页面中提示老师及时下载或保存视频。

## 题目演示生成流程

应用不直接运行模型生成的任意代码。模型只负责输出结构化演示方案，本地模板负责生成 HTML/JS/CSS。

1. 老师输入具体题目。
2. 应用调用文本模型识别题型、已知量、求解目标、步骤和适合的演示方式。
3. 应用校验模型输出。
4. 如果识别为路程、速度、时间类题目，使用高质量路程模板。
5. 如果识别为方程题，使用高质量方程模板。
6. 如果识别为工程、几何等题型，使用简版互动模板。
7. 应用生成独立演示项目目录。
8. 应用启动本地 localhost 服务并打开演示页。
9. 历史记录保存题目、结构化解析、演示页路径和生成时间。

路程模板第一版能力：

- 显示 A/B 两地和距离标注。
- 显示速度、计时器、当前位置。
- 支持开始、暂停、重播。
- 小明或物体按速度沿路线移动。
- 到达终点时展示计算过程和答案。

方程模板第一版能力：

- 从应用题中提取未知量，并用 `x` 或老师可读的变量名表示。
- 展示“找等量关系”的过程，例如总量相等、价格相等、人数相等或距离相等。
- 使用天平、分组卡片或线段图呈现等式两边。
- 支持逐步播放：设未知量、列方程、移项或合并同类项、求解、回代检验。
- 每一步显示对应文字解释，方便老师边播放边讲解。
- 最后展示答案、检验过程和常见易错点。

其他题型简版模板：

- 工程题：进度条、总量、效率、时间。
- 几何题：基础图形、已知量标注、面积或周长计算步骤。

## 本地数据

数据只保存在本机。

建议数据类型：

- 设置：文本 API Key、文本模型名、视频 API Key、视频模型名。
- 教案记录：输入知识点、结构化教案、Markdown、Word 路径、创建时间、更新时间。
- 视频任务：requestId、状态、提示词、脚本、视频 URL、错误原因、创建时间。
- 演示记录：原题、题型、结构化解析、演示页路径、创建时间。

API Key 不应写入前端日志，也不应出现在导出的教案文件中。

## 错误处理

设置错误：

- API Key 缺失时，引导进入设置页。
- 模型名为空时，禁止发起生成。
- 连接测试失败时，展示 HTTP 状态、错误摘要和建议检查项。

文本生成错误：

- JSON 校验失败时，自动重试一次。
- 重试后仍失败时，保存错误摘要和原始输入。
- 页面提供重新生成按钮。

视频生成错误：

- 提交失败时，保留视频脚本和提示词。
- 轮询超时或失败时，显示任务状态和错误原因。
- 允许老师修改提示词后重新生成。

演示页生成错误：

- 模型解析失败时，提示重新生成。
- 模板渲染失败时，保留结构化解析。
- 提供退回简版图文讲解页的按钮。

## 测试与验收

第一版测试重点：

- 教案 JSON schema 校验。
- 教案 Markdown 复制内容完整。
- Word `.docx` 导出结构正确。
- 路程题能抽取距离、速度、求解时间。
- 路程题演示页能开始、暂停、重播，并在终点显示答案。
- 方程题能抽取未知量、等量关系、方程和求解步骤。
- 方程题演示页能逐步展示设未知量、列方程、求解和检验。
- 视频任务能提交、轮询、成功保存 URL、失败保留脚本。
- 设置页能保存、读取、清空和测试两组模型配置。
- 历史记录能恢复教案、视频任务和演示页路径。

手动验收：

- Windows 上启动桌面应用。
- 配置文本模型和视频模型。
- 输入一个知识点生成教案。
- 复制 Markdown。
- 导出 Word。
- 自动提交视频任务并看到状态变化。
- 输入路程题生成互动演示页。
- 输入方程题生成互动演示页。
- 在 localhost 打开演示页并播放动画。

## 第一版成功标准

第一版完成时应满足：

- 老师可以在 Windows 桌面应用中完成一次知识点备课。
- 教案内容完整，有重点、难点、疑点、例题、解法和视频辅助内容。
- 教案可以复制 Markdown，也可以导出 Word。
- 系统可以自动发起视频生成，并允许老师后续改脚本或提示词重新生成。
- 常见路程题可以生成稳定、形象、可交互的 localhost 演示页。
- 常见方程题可以生成稳定、形象、可逐步播放的 localhost 演示页。
- 工程、几何题至少能生成简版互动或图文演示页。
- 所有配置和历史都保存在本机。
